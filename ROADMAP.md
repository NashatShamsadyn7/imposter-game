# 🚀 خطة المستقبل — Imposter Game

> خارطة طريق للتطوير والترقية والانتقال إلى منتج **عالمي (international)** قابل للتوسّع.
> آخر تحديث: 2026-06-06

---

## 🎯 الرؤية (Vision)
تحويل اللعبة من نسخة محلّية (كردي/عربي، تصميم جوال) إلى **منتج عالمي تنافسي** يتحمّل عشرات آلاف اللاعبين المتزامنين، بتجربة آمنة وعادلة وقابلة للتوسّع.

---

## 📊 الوضع الحالي (تقييم سريع)
**التقنيات:** React 18 + Vite + TailwindCSS · Supabase (Auth/DB/Realtime/Edge) · LiveKit (صوت) · Vercel.

| الجانب | الحالة |
|---|---|
| اللغات | كردي (افتراضي) + عربي فقط، عبر `t()` بسيط (مفاتيح كردية) |
| التصميم | جوال أولًا، أُضيف شريط جانبي + توسيع ديسكتوب |
| اللعب | أونلاين (غرف) + محلي + أدوار + بوتات ذكية (Groq) |
| الأمان | ⚠️ الأدوار تُزامَن لكل العملاء (ثغرة غش) · FK محذوفة للبوتات |
| التوسّع | لم يُختبر تحت حمل · لا مراقبة · لا staging |

## 🔴 P0 — حرجة (افعلها أولًا)

### 1. أمان اللعب: إخفاء الأدوار من جهة الخادم (Server-authoritative)
**المشكلة:** `room_players` يحوي عمود `role` ويُرسَل لكل العملاء عبر Realtime. أي لاعب يفتح Network/Console **يرى من هو المحتال** → غشّ كامل، يقتل اللعب التنافسي والمواسم.
**الحل:**
- لا تُرسل `role` للعملاء. خزّنه في جدول منفصل `room_roles` بسياسة RLS تُظهر للّاعب **دوره فقط** (`auth.uid() = user_id`).
- الكشف عند `reveal` عبر RPC `get_my_role(room_id)` يعيد دور الطالب فقط.
- الحلفاء (المحتالون الآخرون) عبر RPC خاص يتحقّق أن الطالب محتال.
- **الأثر:** يجعل المواسم/التصنيف ذا معنى. **بدونه، نظام ELO بلا قيمة.**

### 2. تدوير الأسرار المسرّبة (Rotate leaked secrets)
- مفاتيح Groq/Supabase/Vercel/GitHub ظهرت في سجلّات/محادثات → **اعتبرها مكشوفة**.
- جدّد: `BOT_API_KEY`، Supabase anon/service، GitHub token، أعد فحص `.codex/config.toml`.
- فعّل **Secret Scanning** على GitHub + امنع رفع `.env`.

### 3. سلامة البيانات للبوتات (بدل حذف الـ FK)
**المشكلة:** حذفنا FK من `room_players.user_id` و `votes` و `messages` → ضعف سلامة عام.
**الحل الأنظف:**
- جدول `bots` مستقل (id, room_id, name)، أو
- إبقاء FK وإضافة `bot_id` منفصل، أو
- FK شرطي (`user_id` يقبل null للبوتات + check constraint).
- الهدف: استعادة ضمانات السلامة للّاعبين الحقيقيين.

## 🌍 P1 — التدويل (Internationalization) — جوهر «العالمية»

### 4. نظام i18n حقيقي
**المشكلة:** `t()` الحالي يستخدم نصوصًا كردية كمفاتيح، والترجمة العربية فقط في ملف واحد. غير قابل للتوسّع لعشر لغات.
**الحل:**
- تبنّي مكتبة: **`i18next` + `react-i18next`** (أو `lingui`).
- مفاتيح رمزية: `t('menu.online')` بدل النص الكردي.
- ملفات لغة: `locales/{en,ar,ku,tr,es,fr,de,fa,...}.json`.
- كشف لغة المتصفّح تلقائيًا + مبدّل لغة في الإعدادات.
- دعم **LTR/RTL** ديناميكي (حاليًا RTL ثابت) — `dir` حسب اللغة.
- تنسيق الأرقام/التواريخ عبر `Intl`.

### 5. اللغات ذات الأولوية للإطلاق العالمي
`en` (إلزامي) → `tr` → `es` → `pt` → `fr` → `de` → `fa` → `hi` → `id`.
- **كلمات اللعبة (`words.js`) لكل لغة** — هذا أصعب جزء (الكلمات السرّية يجب أن تكون بلغة اللاعبين). يحتاج مجموعات كلمات منفصلة لكل لغة + اختيار اللغة في إعداد الغرفة.

## 🏗️ P1 — التوسّع والبنية التحتية (Scale to 25k+ concurrent)

### 6. حدود Supabase Realtime
- Realtime له حدود اتصالات/رسائل متزامنة. عند 25k متزامن:
  - راقب **Concurrent connections** و **messages/sec**.
  - قلّل قنوات الاشتراك (حاليًا قناة لكل غرفة — جيد، لكن راقب الـ fan-out).
  - فعّل **Realtime Authorization** (RLS على القنوات).
- ترقية خطة Supabase (Pro/Team) + تفعيل **Read Replicas** للاستعلامات الثقيلة (Leaderboards).

### 7. تخفيف الضغط عن قاعدة البيانات
- **Leaderboards**: حاليًا `season_leaderboard` يمسح `game_results` بالكامل → بطيء عند النمو.
  - حلّ: جدول مُجمَّع `leaderboard_cache` يُحدَّث دوريًا (cron/Edge) أو **Materialized View** تُحدَّث كل دقيقة.
- فهارس (indexes) على الأعمدة الساخنة (موجودة جزئيًا — راجعها).
- **Rate limiting** على RPCs الحسّاسة (`quick_match`, `add_coins`) لمنع الإساءة.

### 8. توسّع البوتات الذكية
- Groq المجاني (30 req/min) لا يكفي عند الحجم. عند النمو:
  - بدّل إلى مزوّد مدفوع (نفس الدالة، متغيّر بيئة واحد).
  - أضف **caching** للأوصاف الشائعة + **حدّ للبوتات لكل غرفة**.
  - فكّر: عند 25k متزامن لن تحتاج بوتات كثيرة (الغرف تمتلئ بلاعبين).

### 9. الأداء (Performance)
- **حزمة LiveKit ~500KB** — مقسّمة (lazy) بالفعل ✅، لكن راجع تحميلها فقط عند الحاجة للصوت.
- صور الكلمات (`WordImage`) — استخدم CDN + أحجام متجاوبة + تحميل كسول.
- **Lighthouse audit** + تقليل حجم الحزمة الأولى.
